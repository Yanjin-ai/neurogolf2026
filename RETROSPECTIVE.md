# 1. 0→1 时间线：七个阶段

## Phase 0 — 调研与真值（Day 0）
- 吃透规则：`cost = 参数元素数 + 中间激活字节数`，`points = 25 − ln(cost)`，
  input/output 张量与节点属性**免费**；banned ops（Loop/Scan/NonZero…）。
- 找到**规则的真值源**：`third_party/arc-gen` 每题的生成器源码 = 该题变换的
  精确定义（不是从样例猜）。这是后来一切"理解规则"工作的地基。
- 排行榜标定：top10%=7271.5、top5%=7305、头部 8060。

## Phase 1 — 本地真值引擎（Day 0-1）
三件套，全项目的基础设施：
- `src/oracle.py`：复刻官方评分（profiler 版）。
- `tools/lean_score.py`：**无 profiler 的静态评分器**——洞察是"静态 shape 网络的
  profiler 观测值 = 静态推断值"，所以 cost 可纯静态算 + 一次 ORT 正确性检查。
  与 oracle 逐分吻合，不写临时文件（profiler 版曾把系统盘写爆）。
- `src/validate.py`：三重门（官方全样例精确匹配 / 数千条 **fresh arc-gen**
  精确匹配 / 约束审计）。fresh 门是防过拟合的命根子。
- **保真度标定**：本地预测 vs Kaggle 实测误差稳定在 **0.4** —— 这个数字让后面
  所有"本地涨了才提交"的决策可信。

## Phase 2 — 手工编译期（Day 1-5，教训期）
- 4 agents 并行攻最差 12 题、LP-conv 编译局部题、逐题时间盒手攻。
- 产出：**净≈0**。血案：task192 的 LP 拟合 conv 本地过 3000 fresh，Kaggle **0 分**
  （d1 提交 7243→7227）。教训固化：**拟合出来的是"可分权重"不是"规则"，
  唯一可信验证是 Kaggle 本身**；自动拟合产物一律默认疑似过拟合。

## Phase 3 — 合并路线（Day 5，最大跃迁）
- 认知转折：公开 notebook 里有大量完整 400 题 bundle，每题独立计分 →
  **逐题取"最便宜且正确"= 合法、安全、可自动化的巨大杠杆**。
- `tools/merge_bundles.py` 两阶段设计：先纯静态排成本（进程内，瞬时），
  再子进程 ORT 验证 cheapest-first（每题只验最便宜的几个）——避免 4600 次
  profiler 子进程写爆磁盘。
- **7268.05 Kaggle 确认，rank 889 → ~299，一步进 top10%。**

## Phase 4 — 机械优化全谱扫描（Day 6，全部归零）
手术 +0.023 / 局部拟合 0 / 结构聚类 6/6 假阳性。三条机械路独立收敛到同一
结论：**公开池的编码层（dtype）已被榨干**。此时的误判：以为算法层也到底了。

## Phase 5 — 算法攻坚与能力标定（Day 6-7）
- task64：读生成器 → 完全理解规则 → 手建电路 **正确 267/0** → 四版 cost-golf
  259642→162442→56886→45259。**但专家网络 10358，公开池 18 个 bundle 全部
  在 10445-13436 窄带里。** 我连最差的公开解都差 3.4×。
- task54：numpy 参考实现 **266 官方 + 3000 fresh 全对**（破解了三个刁钻细节），
  但 ONNX 成本预算 ~60k vs 现网络 25144 → **止损不移植**。
- 能力边界被精确标定：**理解规则不是瓶颈，把逻辑压进专家级张量数才是**（§2.6）。

## Phase 6 — 逆向专家电路（Day 7，真技术库）
把现网络 task64 的 68 个节点全部拆开（§2.5 五技术）。**推翻了"公开 baseline
是 naive 网络"的假设**：它们是手工专家紧电路。也推翻了自己此前"这是不可学的
天赋"的断言——是五个具体、可学、可迁移的技术。

## Phase 7 — 决赛日收割自动化（Day 8）
- 三个没挖过的源：更新版 kernel（+1.7）、**Kaggle datasets**（+0.5，从没搜过）、
  全量 100 kernel + 多版本子目录展开（+0.3）。
- `tools/harvest.py` 循环：扫→拉→合→本地涨≥0.1 才自动提交。
- **终局血案（merge5）**：79 个未审查 bundle 深度合并，本地 7271.06 →
  Kaggle **7254.32**。毒网络 = 过公开样例但挂私有集。**合并的正确性门槛
  （只验公开样例）对未审查来源不够**——task192 教训在 bundle 规模复现。
  幸 Kaggle 按最佳提交计分，merge4 7270.31 战绩无损。

# 2. 技术内核（内化版——每一条都是"我们自己的"了）

## 2.1 成本模型的第一性原理
```
cost   = Σ(中间张量元素数 × dtype字节) + 参数元素数
points = 25 − ln(cost)          ← 对数！砍半只 +0.69，4×压缩才 +1.39
免费项 = input/output 张量、节点属性、Constant
```
三条推论（全部实测验证过）：
1. **能塞进图输出或属性的，别做成中间张量**（OneHot 直接当 output = 免费）。
2. 三个乘数都要压：**元素数**（切片到实际尺寸）、**dtype**（bool/int8 阶梯）、
   **张量个数**（融合，见 2.6）。
3. 对数计分 → 单题优化上限 ~1-2 分 → **战略上批量杠杆 >> 单题英雄主义**。

## 2.2 本地真值引擎
- 静态 shape 网络：cost 可**纯静态**算（shape inference 遍历 value_info），
  与 profiler 逐分一致 → 快 100×、零磁盘。
- 正确性 = 一次 ORT run + `(out>0) == one-hot 输出` 精确比较。
- **跟踪误差 0.4** 是整个决策系统的信任锚——但它只对**经过审查的来源**成立
  （merge5 证明）。

## 2.3 验证体系与过拟合的两次血案
```
三重门：官方全样例精确 → 数千 fresh arc-gen 精确 → 约束审计
```
- **task192（单题）**：LP 拟合过了 fresh 门居然还是挂——因为 fresh 门用同一
  生成器，LP 找到的可分超平面恰好泛化到同分布，但私有集/Linux ORT 环境挂。
  → 修正：**拟合产物永远不可信，只信"实现真规则"的构造性电路**。
- **merge5（bundle 规模）**：合并的门槛只有官方样例（268 条），对大池子里
  的毒网络不设防。→ 修正：**未审查来源必须加 fresh 门或声誉门**
  （高票/有 LB 记录的 bundle 是安全的——它们的分数本身就是私有集验证）。
- 元规则：**验证强度必须匹配来源的可疑度。**

## 2.4 合并 = 选择器：威力与天花板的数学
- 威力：N 个 bundle 逐题取最优 = 每题拿到公开世界的地板 → +25 分一次到手。
- 天花板：**选择器的输出 ≤ 被选集合的逐题并集**。7268-7271 就是公开并集的
  精确值。要超过它必须"创造"——而创造的产出（第 0 节账本）= 0。
- 推论：**合并类策略的边际收益 = 新来源的流入速度**，决赛日收割 = 吃流入。

## 2.5 专家电路五技术（task64 逆向，B11，带方程内化）
1. **结构不变量判别**：`isbox = (count == rowspan×colspan)`——实心矩形的
   面积恒等式当判别器，代替颜色频率长链。泛化：任何"实心 vs 稀疏"判别
   找一个 O(1) 不变量。
2. **ArgMax 直取边界**：`ArgMax(M, axis, select_last_index)` 一个算子出
   每行/列的首/末非背景位置。代替 mask×ramp+ReduceMax 两步。
3. **单区间填充**：每行算 `[lo, lo+wid]` 一个区间，`Sub + LessOrEqual`
   两个算子填出整个方向。代替 4 方向 × 多布尔张量。
4. **算术顶替分支**：`coord + boxMask×100` 把要排除的坐标推出范围——
   用算术代替逻辑门控，少一层 mask。
5. **单 Where 输出**：`Where(fill[1,1,H,W], sel[1,10,1,1], input)` 广播
   选择器直接当图输出（免费）。代替 OneHot 索引网格 + int64。

加上自己实测出的：**Einsum 融合投影** `'bchw,ec->beh'`（选色+降维一步，
900→30）、**OneHot 越界=全零**（padding 哨兵 99）、**MaxPool=形态学膨胀**
（3×3 盖章一个算子）。

## 2.6 图级编译器思维：瓶颈迁移律（本项目最深的技术洞察）
task64 四版实测：**瓶颈随优化迁移——大张量 → dtype → 张量个数**。
- v1 死于 [1,10,30,30] float 大张量（36000/个）
- v2/v3 dtype 压到底后，死于**82 个中间张量 vs 专家 ~30 个**
- 专家的差别不是会更多算子，而是**把逻辑折叠进更少的重算子**：
  我"每步逻辑一个张量"，他们"一个 einsum 干三步"。
- **这本质是编译器/电路设计思维**：目标不是清晰分解，是最小状态周转。
  直接映射到 zero-BP 的状态最小化、LLM graph-schedule、compiler-agent。

## 2.7 SWAR / bit-packing 内核
- 源头思想：一个整数字 = 多个比特槽，**移位 = 空间平移，移位+OR = 传播**。
- 一行 30 布尔 → 1 个 int32：900 字节 → ~120 字节，连通性/flood-fill 的
  理论最优表示（公开网络 task002/286/243 实用）。
- 三个实撞的坑：float 精确整数上限 2^24（30 位要切两段）、移位跨行泄漏
  要 mask、收敛步数要覆盖最坏尺寸。

## 2.8 平台硬边界表（全部实测，设计前查表）
| 算子 | 支持 | 不支持 |
|---|---|---|
| CumSum | int32/int64/float | **uint8/int8/int16**（8 个 CumSum 就锁死 28800） |
| Einsum | int32/float | **uint8** |
| ReduceMax/Min | **uint8** ✅ | — |
| OneHot | 索引 int64、值 float/int64；**索引≥depth→全零** | 索引 int32、值 int8 |
| MaxPool | float/**uint8** ✅ | — |
| ORT 本机坑 | group Conv bias 按组平铺；同进程 Session 污染同构模型（**多模型必须子进程隔离**） | |

## 2.9 逆向工作流（技术库的唯一可靠来源）
```
1. Counter(op_type)            → 用了什么重算子
2. 逐 value_info 元素×字节排序  → 钱花在哪
3. 打印 Einsum equation 属性    → 它怎么缩并/投影
4. 顺 producer 链读 1D→2D 组合  → 算法是什么
```
反面教训：**从生成器源码聚类找"同结构题"= 6/6 假阳性**——判断两题是否同一
计算必须先理解两题的计算，签名替代不了理解。技术库在**电路里**，不在生成器里。

## 2.10 系统工程杂项（血泪清单）
- **磁盘卫生**：profiler json 4600 次评分累积 14G 写爆系统盘 → 无 profiler
  评分器 + 所有临时目录进项目内 `.tmp`。
- **exFAT 三坑**：不支持符号链接（venv 用 `--link-mode=copy`）；wheel 自带
  `._` AppleDouble 文件装不上（预建目标目录/stub 掉 IPython/matplotlib）；
  大簇让 venv 虚占 82G（删 torch/sympy 后 38G）。
- **子进程隔离**：ORT 同进程 Session 污染 → 每模型一个子进程评测。
- **后台任务纪律**：长任务 nohup + 日志 + monitor；合并与收割循环互斥。

# 3. 认知复盘：五个元教训（比技术更值钱）

## 3.1 选择不能超越被选集合
合并、收割、手术、拟合——全是"选择/变换已有解"。它们把我们送到公开并集
（7270），一步都迈不过去。**+27.3 里 0 分来自创造**。要过线必须发明，而发明
需要理解 + 模式库 + 融合直觉——最后一环是我们实测的短板。

## 3.2 别从理论宣布墙——去测量（本项目被打脸三次的错误）
1. 断言"昂贵题贴着成本地板" → 用户逼问 → 排行榜 8060 证明算法层有 792 分余量
2. 断言"einsum 融合是不可学的天赋" → 真去读电路 → 是五个可学技术
3. 断言"手术+0.023 = 无解" → 只是证明了编码层到底，算法层没碰
**规则：每一个"不可能"都必须附带测量。没有测量的墙是自己想象的。**
（对称地：merge5 也教育了反向——没有测量的"安全"同样是想象的。）

## 3.3 估算系统性乐观——先算账再动工
task64 估 ~14k 实测 45k；据此 task54 估 60k 就果断止损（省下几小时注定
失败的移植）。**规则：动工前逐张量列预算；自己的预算 × 惯性乐观系数 ≈ 3。**

## 3.4 理解 ≠ 有竞争力的编码（能力边界的精确坐标）
task54 参考实现 3266 例全对证明"理解规则"满分；task64 差 4.4× 证明
"编码成最小电路"不及格。这个能力叫**图级张量融合直觉**，需要数周刻意练习
（见 `GRAPH_COMPILER_CURRICULUM.md`），不是临场可补。**知道自己缺的具体是
哪块肌肉，是这次比赛最私人的收获。**

## 3.5 地板安全纪律（让所有实验免费）
- 已确认的最佳提交 = 神圣底盘，任何实验产物只有"更便宜**且**过验证"才可能
  被合并选中——失败实验的成本恒为 0 分。
- 关键版本只读锁定（`protected/`），兜底先行再冒险。
- 时间盒 + 分级验收（"及格/良好/优秀"）防止心理过拟合于单一目标。

# 5. 如果重来一次：Day-0 打法（反事实清单）

1. **第一天就把"所有"公开源扫干净**：kernels（全量+按票+按时间）、
   **datasets**（这次 Day-7 才想到，晚了 6 天）、GitHub、HF。合并 = 立即站上
   公开前沿。这次花了 5 天手工编译才想到合并——顺序反了。
2. **建任何电路前先逆向 3 个专家电路**——技术库先行，避免用自己的笨算法
   迭代四版才发现别人的算法根本不同。
3. **每个"机会"先做预算核算**（逐张量列账 ×3 乐观系数），账不过不动工。
4. **验证强度匹配来源可疑度**：高票 bundle 直接合，未审查 bundle 过 fresh 门。
5. 手工攻坚只打"可达下界扫描"确认 gap≥3× 的题，且带时间盒和分级验收。

# 6. 迁移：这套东西对研究主线的价值

- **成本模型下的图设计** ↔ zero-BP 的激活/状态周转最小化：同一个"瓶颈迁移律"
  （大张量→dtype→张量个数）在训练内存优化里逐字成立。
- **逆向工作流**（op 直方图→逐张量成本→equation→producer 链）↔ 任何
  "读懂别人的高效实现"场景：内核、编译 pass、CUDA graph。
