# NeuroGolf 2026 技术总结（可迁移到 ARC-AGI-2 / 后续比赛）

写于 2026-07-08。本文是把这次比赛沉淀下来的、跨比赛可复用的方法学与结论。

## 1. 问题本质的重构

NeuroGolf 表面是"神经网络训练"，实质是 **program-synthesis + 电路 golfing**：
- **不需要推断规则**：google/arc-gen 每题一个 Python 生成器 = 真值规则的可执行形式，且是官方私有集的同分布采样器。michaelhodel/arc-dsl 另有 400 题符号 solver。
- **需要的是**：把已知规则编译成满足静态/无禁用 op/最小 cost 约束的 ONNX，并抗私有集过拟合。

**可迁移教训**：遇到"few-shot 学习"外壳的比赛，先查是否存在生成器/参考实现把"学习"退化成"编译"。这是最大的杠杆。

## 2. 评分器的物理规律（决定一切设计）

`cost = params(元素数) + memory(中间张量字节数)`，`points = max(1, 25 − ln(cost))`。

关键不变量（逆向官方评分器 + 实测得到）：
- **input/output 张量免费，节点属性免费** → 单节点电路 cost = 仅参数，内存为 0。这是所有满分/高分电路的来源。
- **log 尺度**：cost 砍半只 +0.69 分，砍 10 倍才 +2.3 分 → **重设计 ≫ 微调**。
- 中间张量字节 = numel × dtype 宽。省内存的唯一手段：减少中间张量数量、降 dtype（int8/bool）、早 reduce。
- 每个 .onnx 可自选 opset → 老版属性算子（Slice-1/Pad-2/Upsample-7）零参数实现几何变换。

**可迁移教训**：任何"最小化"比赛，先把评分函数当物理定律，实测每种算子/dtype/结构的确切成本，画出"设计阶梯"，再据此选型。不要凭直觉。

## 3. 抗过拟合 = 生命线

私有集用生成器同分布采样。任何"在有限公开样例上正确"的电路都可能在私有集清零。
- 实测：LP 拟合器用 400 样例拟合 → 过 265 官方样例但 609/2000 新样例崩。
- 纪律：**拟合和验证都用 ≥3000 现场生成的新样例**；真正的局部性/规则必须在大样本上成立，官方 ~265 样例会假阳性。

**可迁移教训**：有生成器时，把"泛化"操作化为"在无限新样本上精确"。这比任何正则化都强。

### 3.1 但"本地无限样本"仍不等于 Kaggle 正确（血的教训）

task192 的 [5,5] LP conv 本地过 3000 新样例（17.17 分），提交后 **Kaggle 得 0 分**（d1 从 7243.33→7227.35）。
LP 找到的是"某组能分开当前样本的权重"，不是真规则；私有集/Linux ORT 的某个配置就崩了。
- **推论**：拟合类电路（LP/SGD/2-layer）Kaggle 不可信，必须 Kaggle 亲自确认才敢进最终提交。
- **安全的**：读生成器源码、把**真规则**手写成整数权重电路（margin≥1）。这类才对私有集稳健。
- **可迁移教训**：区分"拟合出通过样本的函数"和"实现产生样本的真规则"。前者过拟合私有分布，后者才泛化。有 oracle 时永远选后者。

## 4. 什么有效 / 什么无效（实测）

| 方法 | 结果 |
|---|---|
| 单节点几何/换色（Transpose/Slice/Gather/Einsum） | 有效但稀少，baseline 大多已拿 |
| 单 Conv 整数权重 LP 拟合 + 核裁剪 | 仅对线性可分局部题有效（11 题中 1 题）；权重上界须≥200（去噪类中心权重需压过邻居计数） |
| 普通两层 Conv（float 中间） | **无效**：float [1,H,30,30] 中间张量太贵，反而更差 |
| int8/QLinearConv 两层 | 理论正解（中间 H×900），未完成实现 |
| 对称核裁剪超越 baseline | 无效：baseline 已做非对称 pad 精调 |
| 评分漏洞 | 已封死，顶分是合法的 |

**可迁移教训**：强公开基线存在时，"容易的自动化"通常已被基线吃掉；边际增益来自基线没覆盖的、需要洞察的结构。要先量化"基线离物理下界还有多远"，只攻还有空间的题。

## 5. 工程坑（ORT/平台）

- ORT group Conv 的 bias/filter 按组平铺（本机 1.24.4）。
- ORT 同进程 session 会污染后续同构模型 → 批量评测必须每模型子进程。
- 提交 zip 文件名必须精确为 `submission.zip`。
- 本地(macOS)与线上(Linux) ORT 行为有差异 → 关键网络用整数权重 + 双侧 margin≥1，避免贴 0 边缘和 ORT 边缘行为。
