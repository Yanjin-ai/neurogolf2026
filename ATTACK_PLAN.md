# 算法攻坚计划：从 7268(top10%) 冲 top5%(+37)

## 0. 核心认知纠正

之前"手术 +0.023 → 无解"的结论**只证明了编码(dtype)到顶，没证明算法到顶**。
排行榜有人做到 8060（比公开 7268 高 792 分）——**算法层有巨大余量，是真能挖的。**

- **成本公式**：`cost = Σ(中间张量元素数 × dtype字节) + 参数`，`points = 25 − ln(cost)`
- **手术只压 dtype 字节**（已到 bool/int8/fp16 地板）
- **算法决定"有多少个中间张量、每个多大"**——这是没被碰过的轴

## 1. "发明更便宜的算法"到底是什么

同一个函数 f，可以用**不同的计算图**实现，成本天差地别：

| 实现方式 | 中间张量 | 成本 |
|---|---|---|
| 笨办法：逐步迭代/逐点循环模拟 | 几十个全网格张量 | 高 |
| 好办法：用对的原语（prefix-scan / JFA / K-模板卷积） | 几个张量 | 低 |

**发明更便宜的算法 = 识别这道题属于哪类计算 → 换上该类的最小原语。**
新算法和现有实现的关系：**函数完全相同，计算图不同**——现网络在"一步步磨"，新网络用一个正确的原语一次算完。

## 2. 计算分类学 + 可借鉴的最小算法（关键！）

每道 ARC 变换属于有限几类，每类有已知最优原语：

| 计算类 | 本质 | 笨实现 | **可借鉴的最小算法** | 典型题 |
|---|---|---|---|---|
| **点/局部** | 邻域重着色 | 多层 conv | 单个 k×k conv | (已便宜) |
| **方向射线/延伸** | 从种子沿轴延伸 | 逐格传播 O(n) | **前缀累积扫描 (cumulative-max, log n 步)** | 054,187,349 |
| **连通性/填充** | 封闭区域、连通块 | 迭代 BFS O(n) | **Jump Flooding (JFA), O(log n) 步** / bit-packing | 002,018,133,285,101 |
| **模板/精灵匹配** | 判断是哪个形状 | 逐位比较 | **K 个固定模板卷积 + argmax** | 364,233,158 |
| **对称/反射** | flip/transpose | 手工搬运 | 原生 Transpose/Flip 算子（近免费） | 054,187 |
| **全局聚合** | 计数/多数色 | 逐格累加 | ReduceSum/ReduceMax | 366 |
| **物体关系** | 排序/配对/匹配 | O(物体²) | 真难，最后攻 | 173,233 |

## 3. 怎么做这个算法研究（每题的循环）

1. **读生成器** `third_party/arc-gen/tasks/task_<hash>.py` → 写出确切规则（我能做）
2. **分类** → 查表 §2 找到该类最小原语
3. **手写 ONNX 电路**：用 `src/graphlib.py` 拼算子，只用最小原语
4. **三重验证**（`src/validate.py`，防过拟合，这是安全命根子）：
   - 全部官方样例精确匹配
   - 2000-5000 条 **fresh arc-gen** 样例精确匹配（私有集代理）
   - 对生成器最坏尺寸/配置推理（如 flood-fill 传播步数够覆盖 30×30）
5. **测成本**（`lean_score.py`）→ 只有比现网络便宜**且**过三重验证才收

## 4. 为什么别人分高（不是运气）

公开 baseline 用的是**通用/半自动编译**（一刀切的笨电路）。8060 的队伍**逐题识别了对的原语**。
差距全在 §2 那张表——他们查了，公开 baseline 没查全。**这是可学的，不是天赋。**
